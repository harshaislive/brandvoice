Prompt Engineering Best Practices

1. Be Specific and Clear

  • Use precise language and avoid ambiguity
  • Define exact requirements and constraints
  • Specify the desired output format

2. Provide Context

  • Include relevant background information
  • Define the target audience clearly
  • Explain the purpose of the content

3. Use Examples

  • Provide good and bad examples
  • Show before/after transformations
  • Demonstrate edge cases

4. Structure Your Prompts

  • Use clear sections and headings
  • Break complex instructions into steps
  • Use bullet points for lists

5. Iterate and Test

  • Test with various input types
  • Refine based on outputs
  • Document what works best

Beforest-Specific Guidelines

  • Calm Self-Assurance: Instruct to avoid urgency or pressure
  • No Superlatives: Explicitly list forbidden words
  • Data-Driven: Request evidence-based language
  • Respect Intelligence: Avoid oversimplification instructions
  • Authenticity: Emphasize natural, honest communication
